# App Spec — Documentation Suite

This suite is the single source of truth for the app. It will back a one-shot "super prompt" build, so **coherence across docs matters more than depth in any single one**. Docs and code are in English; Spanish appears only as UI copy inside the docs.

## Working method

1. **Skeleton first** — all docs exist as outlines with open decisions (current state).
2. **One vertical slice** — take the core journey through every layer to full depth, to stress-test the doc format.
3. **Deepen in rounds** — fill the remaining docs together in passes, with a cross-doc consistency check between rounds.

The suite is a graph, not a pile of prose: every artifact has a stable ID, every fact has exactly one owning doc, and everywhere else it appears as an ID reference. The mechanics — ID prefixes, reference rule, status gates — live in [00-conventions.md](00-conventions.md). Resolved decisions and their rationale live in the [decision log](decision-log.md).

## Deferred (after the vertical slice)

- `SUPER_PROMPT.md` — authority rules, output expectations, and the "unspecified behavior is a blocking decision, never invented" contract.
- `manifest.yaml` — canonical doc order, dependencies, inclusion priority for prompt assembly.
- Full validator — generated traceability matrix and generated index. (A **minimal
  checker exists now**: `scripts/check-docs.py` — frontmatter, local links, comment
  balance, placeholder leaks, README status drift, D-/T- reference resolution. Run
  it before committing doc changes.)
